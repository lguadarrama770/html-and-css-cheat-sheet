What is Git?

Git is a version control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for software development, but can be used to keep track of changes in any files.

  • BASIC GIT CONCEPTS
  • GIT COMMANDS TO KNOW

Basic Git Concepts-

Repository:

  • The overall package or project. There is typically a remote of origin repository, and a local copy of it at certain time on your local machine

    • Branch

  • An extension of a repository. All repositories, or repos, start with a `master` branch. From there, new branches can be created and used for new features. The idea is to keep the master branch clean and working for the final "done" project.

    • Commit:

  • A commit is a change to the repository. For every commit made, there is a record of the state of the repository at that given point in time.

    • Git Commands To Know-

    Changing directories and displaying list of files

  • cd (change directory): Changes the directory or folder
  • cd ~: will take you to the home directory
  • cd ..: moves up one directory
  • ls: lists the files or directories

    • Creating local repository and clone project

  • git init: Creates a new local repository in the current directory
  • git clone:
  • Clones a repository from a remote resource (GitHub, Bitbucket, etc.)
  • Creates a directory inside the current directory, and puts the project in there

    • Creating a branch and switching branches. Also view all branches

  • git checkout -b {branch name}: shorthand command to create and switch to a new branch
  • git branch {branch name}: creates a new branch
  • git checkout {branch name}: switches to the branch
  • git branch -a: displays all branches, both local and remote, that are available

    • Checking the status of a repository

  • git status: (RUN A LOT)
  • Tells what files have been added, deleted, and modified
  • Informs if the local branch is behind the remote version or if the version is ahead of the remote repository

    • Adding a file, directory, or othe

  • git add {file, directory, or, other}: stages the changes to be committed
  • git branch .: Stages all changes from the directory

    • Committing changes to the repository

  • git commit -m”(commit message)”: This commits the added changes locally and attaches the commit message to the commit
  • Commit messages should be short but descriptive of all the changes made
  • Committing does not push changes to the remote repository.

    • Sending commits to the repository

  • git push: Takes the locally committed changes for a given branch and applies them to the local version
  • git push -u origin {branch name}: This command is used for pushing to the remote repository with the specific branch name

    • Pulling from a repository

  • git pull -- Takes the changes made to the remote repository and applies them to the local version

    • Creating directories and delete

  • mkdir <folder name>
  • rmdir <folder name>
